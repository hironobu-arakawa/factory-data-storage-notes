# 保存場所は、責任の置き場所である

このドキュメントは、  
「どこに保存するか」が  
**性能やコストの話ではなく、責任の話である**ことを明確にします。

---

## 保存すると、責任は移動する

データを保存するという行為は、

- 値を残すこと
- 時系列を保持すること

ではありません。

保存とは、

> **「このデータは、ここまで正しい」と宣言すること**

です。

そしてその宣言は、  
**保存した場所に責任を移します。**

---

## DB に保存したとき

### 何が起きるか

DB に入れた瞬間、次が起きます。

- スキーマが契約になる
- 正しさが暗黙に保証されたと見なされる
- 後段は「疑わなくてよい前提」で使い始める

特に RDB では、

> DB = 正本（canonical）

という誤解が極めて強く働きます。

---

### 固定される責任

- 値の正しさ
- 欠損や異常の扱い
- 更新・修正の可否

これらはすべて  
**DB 管理側の責任**として扱われます。

その覚悟がないなら、  
DB に入れてはいけません。

---

## TSDB に保存したとき

### 一見、中立に見えるが…

TSDB は、

- append-only
- 高速
- 粒度が細かい

ため、  
「とりあえず入れる」誘惑が強い場所です。

しかし TSDB に入れた瞬間：

- 時刻精度が契約になる
- 観測値が事実扱いされる
- 補正は“例外”になる

---

### 固定される責任

- 時刻の正しさ
- 欠損の扱い
- 外れ値の意味

TSDB は  
**「事実らしく見せる力」**が強い分、  
誤った責任固定を起こしやすい保存先です。

---

## Object Storage に保存したとき

### 責任を保留できる唯一の場所

Object Storage は、

- raw
- 非構造
- 後解釈前提

という性質を持ちます。

ここに保存する行為は、

> 「まだ Fact にしない」

という意思表示でもあります。

---

### 固定される責任（限定的）

- 消さない責任
- 改変しない責任

一方で：

- 意味
- 正しさ
- 利用方法

は **まだ固定されません**。

---

## 保存場所ごとの責任まとめ

| 保存先 | 固定されるもの | 固定されないもの |
|------|---------------|----------------|
| DB | 意味 / 正しさ / 契約 | 再解釈 |
| TSDB | 時刻 / 観測事実 | 文脈 |
| Object | 存在 / 原本 | 意味 / 利用 |

保存先の違いは、  
**責任の固定範囲の違い**です。

---

## よくある致命的な誤り

### 「後で DB に移せばいい」

一度 Object → DB に移した瞬間、

- 責任が確定
- 解釈の余地が消える

「後で決める」は通用しません。

---

### 「TSDB だから安全」

TSDB は安全ではありません。  
**速く固定する装置**です。

---

## まとめ

- 保存場所は技術選択ではない
- 保存場所は責任宣言である
- DB / TSDB / Object は責任の強さが違う
- 覚悟のない場所に保存してはいけない

---

保存とは、  
**「どこで責任を引き受けるか」を決める行為**です。

この問いに答えられない限り、  
保存先を選んではいけません。
